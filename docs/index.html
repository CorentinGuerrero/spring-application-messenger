Overview

Spring Application Messenger

Spring-native application messaging for commands, queries, events, middleware, transports, outbox publishing, and test-friendly handlers.

version 0.1.0-SNAPSHOT java 21+ spring boot 3.5.x namespace io.github.applicationmessenger

The library is inspired by Symfony Messenger, but stays deliberately smaller and Spring-native. Application code talks to CommandBus, QueryBus, and EventBus. Routing decides whether a command or event is handled locally, sent to a broker, or stored in a JDBC outbox before being published.

Queries are synchronous and in-process by design. Broker routing applies to commands and events.

Installation

Artifacts are not published yet. These are the intended coordinates for a future release. In the repository today, modules are consumed as local Gradle projects. 

dependencies {
    implementation "io.github.applicationmessenger:spring-application-messenger-starter:0.1.0"

    implementation "io.github.applicationmessenger:messenger-transport-rabbitmq:0.1.0"
    implementation "io.github.applicationmessenger:messenger-transport-kafka:0.1.0"

    implementation "io.github.applicationmessenger:messenger-transport-jdbc-outbox:0.1.0"

    testImplementation "io.github.applicationmessenger:messenger-test:0.1.0"
}

Quickstart

Define a message, add a handler, and dispatch through the matching bus. 

public record RegisterUser(String email)
    implements Command<UserId> {
}

@CommandHandler
public class RegisterUserHandler {
    private final UserRepository repository;
    private final EventBus eventBus;

    public UserId handle(RegisterUser command) {
        User user = User.register(command.email());
        repository.save(user);
        eventBus.publish(new UserRegistered(user.id(), user.email()));
        return user.id();
    }
}

UserId userId = commandBus.dispatch(new RegisterUser("john@example.com"));

Configuration

This is the shape of the complete example configuration: one sync command, one RabbitMQ command, one Kafka command, and one event stored in the JDBC outbox. 

messenger:
  validation:
    enabled: true
  events:
    error-strategy: fail-fast
  routing:
    commands:
      RegisterUser: sync
      SendWelcomeEmail: rabbitmq
      RebuildUserSearchIndex: kafka
    events:
      UserRegistered: pg-outbox
  transports:
    rabbitmq:
      exchange: messenger.example
      routing-keys:
        SendWelcomeEmail: mail.welcome
      consumer:
        enabled: true
        queues: messenger.example.mail
    kafka:
      topics:
        RebuildUserSearchIndex: messenger.example.user-index
      consumer:
        enabled: true
        topics: messenger.example.user-index
        group-id: messenger-example
    jdbc-outbox:
      initialize-schema: true
      table-name: messenger_outbox
      publisher:
        enabled: false
        target-transport: sync

Messages

Type Bus Handler Count Result Purpose
Command<R> CommandBus Exactly one Optional result Express an intention to modify state.
Query<R> QueryBus Exactly one Required result Read application state synchronously.
Event EventBus Zero, one, or many No result Describe a business fact that already happened.

Handlers

Handlers are regular Spring beans annotated with Messenger annotations. The scanner validates method signatures during startup and fails fast on invalid configuration. 

@CommandHandler
public class SendWelcomeEmailCommandHandler {
    public Void handle(SendWelcomeEmail command) {
        // send email
        return null;
    }
}

@EventHandler
public class SendWelcomeEmailHandler {
    public void handle(UserRegistered event) {
        // react to event
    }
}

Middleware

Middleware runs around dispatch before the final local handler or transport. It is useful for validation, transactions, logging, authorization, observability, retry, and custom policies. 

public interface MessageMiddleware {
    Object invoke(MessageEnvelope envelope, MessageHandler next);
}

Routing

Routing supports simple class names and fully qualified class names. Use fully qualified names when two messages share the same simple name.

syncLocal in-process dispatch.
rabbitmqRoute to RabbitMQ exchange/routing key.
kafkaRoute to Kafka or Redpanda topic.
redisRoute to Redis Streams.
pg-outboxStore in JDBC outbox table for later publishing.

TransportMessage

Broker connectors send a TransportMessage. Consumers reconstruct the payload from messageType, then dispatch it through the incoming dispatcher. 

{
  "messageType": "com.acme.UserRegistered",
  "busType": "EVENT",
  "transportName": "kafka",
  "payload": {
    "userId": "4b7b...",
    "email": "john@example.com"
  },
  "metadata": {
    "messageId": "...",
    "correlationId": "...",
    "causationId": null,
    "createdAt": "...",
    "headers": {}
  }
}

Spring Boot Example

The repository includes spring-messenger-example, a complete app using sync, outbox, RabbitMQ, and Kafka/Redpanda in one configuration. 

./gradlew :spring-messenger-example:bootRun

./gradlew :spring-messenger-example:test --tests '*ContainerTest'

Testing

The messenger-test module gives reusable utilities for app tests: fake command/query buses, recording event bus, event assertions, and handler helpers. 

RecordingEventBus eventBus = new RecordingEventBus();
RegisterUserHandler handler = new RegisterUserHandler(repository, eventBus);

handler.handle(new RegisterUser("john@example.com", "John"));

assertThat(eventBus).hasPublished(UserRegistered.class);
