Connector

JDBC Outbox

The JDBC outbox stores messages in the same database transaction as business changes. A publisher later forwards pending rows to a target transport such as sync, RabbitMQ, Kafka, or Redis.

Install

dependencies {
    implementation "io.github.applicationmessenger:messenger-transport-jdbc-outbox:0.1.0"
    implementation "org.springframework.boot:spring-boot-starter-jdbc"
}

Configuration

messenger:
  routing:
    events:
      UserRegistered: pg-outbox
  transports:
    jdbc-outbox:
      enabled: true
      initialize-schema: true
      table-name: messenger_outbox
      initial-status: PENDING
      publisher:
        enabled: true
        batch-size: 50
        max-attempts: 3
        poll-interval: 5s
        retry-delay: 30s
        target-transport: kafka
        routes:
          UserRegistered: kafka

Connector Options

enabledEnables the outbox transport. Defaults to true when JdbcTemplate is present.
initialize-schemaCreates the outbox table automatically. Default: true.
table-nameOutbox table name. Default: messenger_outbox.
initial-statusStatus assigned to newly stored rows. Default: PENDING.
publisher.enabledStarts the scheduled publisher.
publisher.batch-sizeMaximum pending messages processed per poll. Default: 50.
publisher.max-attemptsAttempts before final failure. Default: 3.
publisher.target-transportDefault transport used by the publisher.
publisher.routesPer-message override from message type to target transport.

Flow

1. HandlerSaves business state and publishes an event.
2. OutboxStores the event as a row in the same transaction.
3. CommitBusiness data and message become durable together.
4. PublisherForwards pending rows and marks them published or failed.

Schema

The connector owns a stable outbox table and can initialize it automatically. Applications should not need to hand-write the table in normal usage.

Known Limitations

  • No broker-level delivery guarantee beyond the selected target transport.
  • No admin UI for failed messages yet.
  • Payload serialization is currently Jackson-based.
