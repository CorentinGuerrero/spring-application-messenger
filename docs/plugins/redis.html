Connector

Redis Streams

The Redis connector routes commands or events to Redis Streams and can run a managed consumer-group listener that ACKs messages after local handler dispatch.

Install

dependencies {
    implementation "io.github.applicationmessenger:messenger-transport-redis:0.1.0"
}

Configuration

spring:
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}

messenger:
  routing:
    commands:
      RebuildSearchIndex: redis
  transports:
    redis:
      enabled: true
      stream-prefix: messenger
      streams:
        RebuildSearchIndex: messenger.search.commands
      consumer:
        enabled: true
        streams:
          - messenger.search.commands
        group: messenger
        consumer-name: worker-1
        poll-timeout: 1s

Connector Options

enabledEnables the Redis transport. Defaults to true when the module and RedisTemplate are present.
stream-prefixPrefix for generated streams when no explicit stream is configured. Default: messenger.
streamsMap of message simple name or fully qualified name to Redis stream.
consumer.enabledStarts the managed Redis Streams listener.
consumer.streamsStreams consumed by the listener.
consumer.groupRedis consumer group. Default: messenger.
consumer.consumer-nameConsumer name used in the group. Default: messenger-worker.
consumer.poll-timeoutBlocking poll timeout. Default: 1s.

Usage Example

public record RebuildSearchIndex(String indexName)
    implements Command<Void> {
}

@CommandHandler
public class RebuildSearchIndexHandler {
    public Void handle(RebuildSearchIndex command) {
        // rebuild index
        return null;
    }
}

Managed Listener

The managed listener creates or uses a Redis consumer group, reads configured streams, converts TransportMessage payloads back to Java messages, dispatches handlers, and ACKs successfully processed entries.

Known Limitations

  • No pending-entry recovery policy yet.
  • No DLQ abstraction yet.
  • Serialization is currently Jackson-based.
