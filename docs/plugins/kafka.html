Connector

Kafka / Redpanda

The Kafka connector routes commands or events to Kafka-compatible topics. It is built on Spring Kafka, so broker connection, SSL, SASL, producer, and consumer settings come from normal spring.kafka.* configuration.

Install

dependencies {
    implementation "io.github.applicationmessenger:messenger-transport-kafka:0.1.0"
}

Configuration

spring:
  kafka:
    bootstrap-servers: ${KAFKA_BOOTSTRAP_SERVERS:localhost:9092}
    producer:
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
    consumer:
      auto-offset-reset: earliest
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: "*"
        spring.json.value.default.type: io.github.applicationmessenger.messenger.transport.TransportMessage

messenger:
  routing:
    commands:
      RebuildUserSearchIndex: kafka
  transports:
    kafka:
      enabled: true
      topics:
        RebuildUserSearchIndex: messenger.example.user-index
      consumer:
        enabled: true
        topics: messenger.example.user-index
        group-id: messenger-example

Connector Options

enabledEnables the Kafka transport. Defaults to true when the module and KafkaTemplate are present.
topic-prefixPrefix for generated topics when no explicit topic is configured. Default: messenger.
topicsMap of message simple name or fully qualified name to topic.
consumer.enabledStarts the Kafka listener for inbound messages.
consumer.topicsComma-separated topic list consumed by the Messenger listener.
consumer.group-idKafka consumer group used by the Messenger listener. Default: messenger.

Usage Example

public record RebuildUserSearchIndex(UserId userId)
    implements Command<Void> {
}

@CommandHandler
public class RebuildUserSearchIndexHandler {
    public Void handle(RebuildUserSearchIndex command) {
        // rebuild index
        return null;
    }
}

commandBus.dispatch(new RebuildUserSearchIndex(userId));

SSL / SASL

The connector does not invent a separate security model. Use Spring Kafka properties such as spring.kafka.properties.security.protocol, sasl.mechanism, sasl.jaas.config, and SSL truststore/keystore settings.

Known Limitations

  • No DLQ abstraction yet. Use Spring Kafka error handlers and broker policies.
  • Transport message schema is currently Java/Jackson oriented.
  • No schema registry integration yet.
