Connector

RabbitMQ

The RabbitMQ connector routes commands or events to an AMQP exchange using Spring AMQP. When the consumer is enabled, incoming TransportMessage payloads are converted back to Java messages and dispatched to local Messenger handlers.

Install

dependencies {
    implementation "io.github.applicationmessenger:messenger-transport-rabbitmq:0.1.0"
}

The connector uses Spring Boot RabbitMQ configuration, so SSL, credentials, host, port, virtual host, and connection options come from spring.rabbitmq.*.

Configuration

spring:
  rabbitmq:
    host: ${RABBITMQ_HOST:localhost}
    port: ${RABBITMQ_PORT:5672}
    username: ${RABBITMQ_USERNAME:guest}
    password: ${RABBITMQ_PASSWORD:guest}

messenger:
  routing:
    commands:
      SendWelcomeEmail: rabbitmq
  transports:
    rabbitmq:
      enabled: true
      exchange: messenger.example
      routing-keys:
        SendWelcomeEmail: mail.welcome
      consumer:
        enabled: true
        queues: messenger.example.mail

Connector Options

enabledEnables the RabbitMQ transport. Defaults to true when the module and RabbitTemplate are present.
exchangeAMQP exchange used for outbound messages. Default: messenger.
routing-key-prefixPrefix for generated routing keys when no explicit key is configured. Default: messenger..
routing-keysMap of message simple name or fully qualified name to routing key.
consumer.enabledStarts the Rabbit listener for inbound messages.
consumer.queuesComma-separated queue list consumed by the Messenger listener.

Usage Example

public record SendWelcomeEmail(UserId userId, String email)
    implements Command<Void> {
}

@CommandHandler
public class SendWelcomeEmailCommandHandler {
    public Void handle(SendWelcomeEmail command) {
        // send email
        return null;
    }
}

commandBus.dispatch(new SendWelcomeEmail(userId, "john@example.com"));

Consumer Behavior

The Rabbit listener receives a TransportMessage, resolves messageType, converts the payload with Jackson, then calls IncomingMessageDispatcher. The dispatched message reaches the same handler method as a local dispatch.

Known Limitations

  • No built-in DLQ policy yet. Configure DLQ behavior through Spring AMQP/RabbitMQ infrastructure.
  • No framework-level retry strategy yet for failed inbound messages.
  • Serialization is currently Jackson-based.
